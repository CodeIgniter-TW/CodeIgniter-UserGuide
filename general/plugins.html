外掛模組(Plugins)

外掛模組(Plugins) 功能與 補助函數幾乎相同。主要的差別是外掛模組(plugins)通常提供單一函數，而補助函數(helpers)則為多個函數組合。補助函數(helpers)為部份核心系統，外掛模組則建立與分享自我們社群。

外掛模組(Plugins) 應儲存於 system/plugins 目錄底下，或者是你可以在 application 目錄中，建立名稱為 plugins 的目錄，將外掛模組儲存於該處。CodeIgniter 會優先察看 system/application/plugins，若發現無目錄存在的時，則會改成察看 global 的 system/plugins 目錄。

載入外掛模組

只用下列的函數來載入外掛模組相當簡單:

$this->load->plugin('name');

name 為外掛模組的名稱，而不需要附加 .php 的副檔名或是 "plugin" 的部份。

假設要載入 Captcha 外掛模組，他的檔案名稱為 captcha_pi.php，_pi.php 的部份可以省略掉，所以只需要載入 captcha 即可:

$this->load->plugin('captcha');

外掛模組可以隨意的由 控制器(controller) 函數或是由 View files所載入，雖然這不是個好例子。只要你在使用前載入即可。你可以在你的控制器建構子(controller constructor)載入外掛模組，接者他們即可自動地被所有函數所用, 或是你可以在某個需要外掛模組的函數中，將它載入近來使用。

注意: 上述外掛模組載入函數並不會回傳質，所以不用給予它變數。就把它顯示出來即可。

載入多個外掛模組(Plugins)

假如需要載入更多的外掛模組到 array 的話，你可以這麼做:

$this->load->plugin( array('plugin1', 'plugin2', 'plugin3') );

自動載入外掛模組(Plugins)

如果發現你的 application 需要全域(globally)使用某個外掛模組的話，你可以告訴 CodeIgniter 在系統啟動時，將那個模組自動載入近來。你可以開啟 application/config/autoload.php 的檔案，加入你需要的外掛模組到 autoload array 裡頭。

使用外掛模組(Plugin)

只要你載入了外掛模組後，你可以用標準的 PHP 函數來呼叫使用它。
