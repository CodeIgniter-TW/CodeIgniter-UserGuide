補助函數(Helper Functions)

補助函數，as the name suggests，help you with tasks. Each helper file is simply a collection of functions in a particular category. There are URL 補助函數，that assist in creating links，there are Form 補助函數 that help you create form elements，Text 補助函數 perform various text formatting routines, Cookie 補助函數 set and read cookies，File 補助函數 help you deal with files，etc.

Unlike most other systems in CodeIgniter，補助函數 are not written in an Object Oriented format. They are simple，procedural functions. Each helper function performs one specific task，with no dependence on other functions.

CodeIgniter does not load 補助函數 Files by default，so the first step in using a 補助函數 is to load it. Once loaded，it becomes globally available in your controller and views.

補助函數 are typically stored in your system/helpers，or system/application/helpers directory. CodeIgniter will look first in your system/application/helpers directory. If the directory does not exist or the specified helper is not located there CI will instead look in your global system/helpers folder.

載入補助函數

Loading a helper file is quite simple using the following function:

$this->load->helper('name');

Where name is the file name of the helper，without the .php file extension or the "helper" part.

For example，to load the URL 補助函數 file，which is named url_helper.php，you would do this:

$this->load->helper('url');

A helper can be loaded anywhere within your controller functions (或說在你的 View files 裡頭，although that's not a good practice), as long as you load it before you use it. You can load your helpers in your controller constructor so that they become available automatically in any function，or you can load a helper in a specific function that needs it.

注意: The 補助函數 loading function above does not return a value，so don't try to assign it to a variable. Just use it as shown.

載入多個補助函數

If you need to load more than one helper you can specify them in an array，like this:

$this->load->helper( array('helper1'，'helper2'，'helper3') );

自動載入補助函數

If you find that you need a particular helper globally throughout your application，you can tell CodeIgniter to auto-load it during system initialization. This is done by opening the application/config/autoload.php file and adding the helper to the autoload array.

使用補助函數

Once you've loaded the 補助函數 File containing the function you intend to use，you'll call it the way you would a standard PHP function.

For example，to create a link using the anchor() function in one of your view files you would do this:

<?php echo anchor('blog/comments'，'Click Here');?>

Where "Click Here" is the name of the link，and "blog/comments" is the URI to the controller/function you wish to link to.

"擴充" 補助函數

To "extend" 補助函數，create a file in your application/helpers/ folder with an identical name to the existing 補助函數，but prefixed with MY_ (this item is configurable. See below.).

If all you need to do is add some functionality to an existing helper - perhaps add a function or two，or change how a particular helper function operates - then it's overkill to replace the entire helper with your version. In this case it's better to simply "extend" the 補助函數. The term "extend" is used loosely since 補助函數 functions are procedural and discrete and cannot be extended in the traditional programmatic sense. Under the hood，this gives you the ability to add to the functions a 補助函數 provides, or to modify how the native 補助函數 functions operate.

For example，to extend the native Array 補助函數 you'll create a file named application/helpers/MY_array_helper.php，and add or override functions:

// any_in_array() is not in the Array 補助函數，so it defines a new function
function any_in_array($needle，$haystack)
{
    $needle = (is_array($needle)) ? $needle : array($needle);

    foreach ($needle as $item)
    {
        if (in_array($item，$haystack))
        {
            return TRUE;
        }
        }

    return FALSE;
}

// random_element() is included in Array 補助函數，so it overrides the native function
function random_element($array)
{
    shuffle($array);
    return array_pop($array);
}

自訂子類別的前置字串

The filename prefix for "extending" 補助函數 is the same used to extend libraries and Core classes. To set your own prefix，open your application/config/config.php file and look for this item:

$config['subclass_prefix'] = 'MY_';

Please note that all native CodeIgniter libraries are prefixed with CI_ so DO NOT use that as your prefix.
